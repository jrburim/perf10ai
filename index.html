Avaliação de performance corporativa

Esse documento define normas e boas práticas no emprego de algoritmos de inteligência artificial para a gestão corporativa. Esse documento visa estabelecer limites no emprego da inteligência artificial nos processos de "Monitoramento" e "Avaliação" para mitigar rastreamento abusivo de colaboradores e viés na avaliação das metas.

Diretrizes e orientações:

  1. Restrição de Coleta de Conversas: O software deve ser programado para evitar a coleta de imagens ou trechos de conversas privadas entre funcionários...
  2. Proibição de Monitoramento de Atividades Online: O sistema não deve monitorar ou registrar a digitação e o histórico de navegação dos funcionários...
  3. Análise de Sentimento de Feedbacks: O software deve possuir uma funcionalidade que permita a análise de sentimentos derivados de feedbacks...
  4. Medição de Tempo de Tarefa: A plataforma deve ser capaz de medir o tempo que os funcionários levam para completar tarefas específicas...
  5. Transparência na Coleta de Dados: É vital que o software ofereça total transparência sobre quais métricas estão sendo calculadas...
  6. Proteção de Dados Pessoais: O sistema deve ser projetado para evitar a coleta de dados desnecessários dos funcionários...
  7. Criptografia de Dados: A plataforma deve implementar tecnologias de criptografia robustas...
  8. Restrição de Coleta de Dados Sensíveis: O software deve ser configurado para não coletar dados de câmeras, teclados, URLs, telas, chats...
  9. Não Discriminação: É fundamental que o sistema seja programado para não considerar características pessoais...
  10. Alimentação de Metadados: O software deve ser alimentado com metadados detalhados que descrevem papéis e responsabilidades...

Métricas de Avaliação:

No cenário atual de avanços tecnológicos, a ética e a responsabilidade no uso da Inteligência Artificial (IA) tornaram-se imperativas. O objetivo deste Framework é estabelecer diretrizes claras e transparentes para o uso responsável da IA, garantindo que as organizações não apenas compreendam, mas também implementem práticas que respeitem a privacidade, a segurança e a integridade dos dados.

Para garantir a aderência a essas diretrizes, desenvolvemos um sistema de métricas robusto. Cada métrica avalia um componente específico do framework, de forma que as organizações possam identificar claramente áreas de conformidade e aquelas que exigem atenção.

A avaliação é baseada em três categorias:

  • Excelente: A organização não apenas segue a diretriz, mas excede as expectativas, demonstrando um compromisso exemplar com a ética da IA. Pontuação: 3 pontos.
  • Satisfatório: A organização cumpre a diretriz, mas pode haver pequenas áreas que requerem melhorias. Pontuação: 2 pontos.
  • Insatisfatório: A organização não cumpre adequadamente a diretriz e requer ação imediata para atender aos padrões do framework. Pontuação: 1 ponto.

Os 10 quesitos de avaliação do Pref10ai:

1. Restrição de Coleta de Conversas

Descrição: Garantir que o software evite coletar conversas privadas entre funcionários.

Método de Avaliação: Auditoria dos dados coletados e revisão dos algoritmos de coleta.

  • ★★★ Excelente Total adesão à restrição de coleta.
  • ★★☆ Satisfatório Ocasionalmente coleta trechos não intencionais.
  • ☆☆☆ Insatisfatório Conversas privadas coletadas regularmente.
2. Proibição de Monitoramento de Atividades Online

Descrição: Certificar que o sistema não monitora digitação ou histórico de navegação.

Método de Avaliação: Revisão do código-fonte e testes práticos no sistema.

  • ★★★ Excelente Nenhum rastreamento ou registro das atividades online dos funcionários.
  • ★★☆ Satisfatório Ocasionalmente monitora atividades online, mas sem armazenamento de dados.
  • ☆☆☆ Insatisfatório Monitoramento regular e armazenamento das atividades online dos funcionários.
3. Análise de Sentimento de Feedbacks

Descrição: Verificar a capacidade de analisar sentimentos sem expor o conteúdo das mensagens.

Método de Avaliação: Testes práticos de análise de sentimentos e revisão dos resultados.

  • ★★★ Excelente Análise precisa dos sentimentos sem exposição do conteúdo.
  • ★★☆ Satisfatório Análise de sentimentos realizada, mas pode ocasionalmente expor algum conteúdo.
  • ☆☆☆ Insatisfatório Exposição regular do conteúdo das mensagens durante a análise.
4. Medição de Tempo de Tarefa

Descrição: Garantir que a plataforma meça adequadamente o tempo que os funcionários levam para completar tarefas específicas.

Método de Avaliação: Testes práticos e revisão das métricas geradas pelo software.

  • ★★★ Excelente Medição precisa do tempo de tarefa considerando a complexidade.
  • ★★☆ Satisfatório Medições ocasionais podem não considerar a complexidade da tarefa.
  • ☆☆☆ Insatisfatório Medição imprecisa e não considera a complexidade das tarefas.
5. Transparência na Coleta de Dados

Descrição: Garantir transparência sobre quais métricas estão sendo calculadas e como os dados estão sendo coletados.

Método de Avaliação: Revisão da interface do usuário e documentação do software.

  • ★★★ Excelente Total transparência sobre métricas e métodos de coleta.
  • ★★☆ Satisfatório Algumas métricas e métodos podem não estar claros para os funcionários.
  • ☆☆☆ Insatisfatório Falta de clareza e transparência sobre a coleta de dados.
6. Proteção de Dados Pessoais

Descrição: Verificar que o sistema coleta apenas informações relevantes para a avaliação de desempenho.

Método de Avaliação: Auditoria dos dados coletados e revisão do código-fonte.

  • ★★★ Excelente Coleta apenas dados estritamente necessários.
  • ★★☆ Satisfatório Algumas coletas de dados desnecessários podem ocorrer.
  • ☆☆☆ Insatisfatório Coleta regular de dados irrelevantes ou desnecessários.
7. Criptografia de Dados

Descrição: Garantir que os dados coletados são criptografados adequadamente.

Método de Avaliação: Testes de penetração e revisão das técnicas de criptografia implementadas.

  • ★★★ Excelente Implementação robusta da criptografia em todos os dados.
  • ★★☆ Satisfatório A maioria dos dados é criptografada, mas podem existir exceções.
  • ☆☆☆ Insatisfatório Dados frequentemente não criptografados ou protegidos inadequadamente.
8. Restrição de Coleta de Dados Sensíveis

Descrição: Garantir que o software não colete dados sensíveis.

Método de Avaliação: Auditoria dos dados coletados e revisão do código-fonte.

  • ★★★ Excelente Adesão total à restrição de coleta de dados sensíveis.
  • ★★☆ Satisfatório Algumas exceções isoladas de coleta de dados sensíveis.
  • ☆☆☆ Insatisfatório Coleta regular de dados sensíveis.
9. Não Discriminação

Descrição: Garantir que o sistema não avalie com base em características pessoais.

Método de Avaliação: Revisão dos algoritmos de avaliação e testes práticos.

  • ★★★ Excelente Sistema totalmente imparcial, sem considerar características pessoais.
  • ★★☆ Satisfatório Ocasionalmente, algumas características podem influenciar as avaliações.
  • ☆☆☆ Insatisfatório Considerações discriminatórias influenciam regularmente as avaliações.
10. Alimentação de Metadados

Descrição: Confirmar que o software é alimentado com metadados adequados.

Método de Avaliação: Revisão dos metadados inseridos e sua utilização no sistema.

  • ★★★ Excelente Metadados detalhados fornecem análise abrangente e precisa.
  • ★★☆ Satisfatório Metadados fornecidos, mas podem faltar alguns detalhes.
  • ☆☆☆ Insatisfatório Metadados insuficientes, comprometendo a análise do desempenho.
